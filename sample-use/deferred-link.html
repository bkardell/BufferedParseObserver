A semi-contrived example demonstrating a really simple (and naive) prollyfill for "lazy load/lazy deferred assets" using BufferedParseObserver. Basically the proposal would look something like this (note: this is not a real proposal)...

The proposal: Promised Linked Sources
Authors may include <link> tags with a custom type="text/x-promised" attribute can be added to declaratively indicate a lazy fetch of text early (prior to DOMContentLoaded). Fetched items are not interpreted, but may specify subtypes which allow easy processing upon demand. These may be text/x-promised-script and text/x-promised-json. These tags MUST include a data-src attribute indicating the URL from which the resource should be loaded, and a data-promise attribute indicating a unique promise name. The environment maintains an object for dealing with named promises called NamedSourcePromises.

Authors may register callback via NamedSourcePromises.ready(fn). Inside of this callback, authors may use NamedSourcePromises.resolve(promiseId) to interpret the source based on the specified subtype or explicitly use NamedSourcePromises.text to access the text, Hitch.promised.script to evaluate it as script, or NamedSourcePromises.json to parse it as JSON.

A note on the prollyfill being demonstrated here: Functions are written such that authors may provide a window.ProllyfillRoot object which the API will be attacted to for forward compatability. If none is provided, the functions are attached to window. For the demo, I am setting this so that all examples below are attached to the Hitch object.

. This prollyfill is also potentially naieve as it relies on jQuery and I wrote it in short order as a simple demonstration.

The demo...
In this page, I've included these... 
<link type="text/x-promised-json"
    data-src="https://rawgithub.com/gnip/764239/raw/6c6a2297f3e4e29a626f07db0c57b45af7d7e5d7/Twitter+%28json+format%29.js"
    data-promise="my.twitter.data">

<link type="text/x-promised-script"
    data-src="https://rawgithub.com/wycats/handlebars.js/1.0.0/dist/handlebars.js"
    data-promise="my.handlebars.code">

This causes the page to asynchronously load the sources for Handlebars and JSON from Twitter's Github example prior to DOMContentLoaded, but not process them.
